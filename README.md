# Linux Security Labs

Hands-on Linux administration and security hardening documentation. Each lab includes commands, explanations, and security context.

## Labs

| Lab | Topic | Status |
|---|---|---|
| 01-user-management | Users, groups, sudo, password policies | Planned |
| 02-file-permissions | chmod, chown, SUID/SGID, ACLs | Planned |
| 03-ssh-hardening | sshd_config, key auth, fail2ban | Planned |
| 04-log-analysis | auth.log, syslog, journalctl | Planned |
| 05-firewall-basics | ufw, iptables, default policies | Planned |

## Environment

- OS: Ubuntu 22.04 LTS
- All labs performed in isolated virtual machines

## Usage

Each lab folder contains a `README.md` with objectives, commands, and explanations. Commands are documented with context — not just how to run them, but why they matter for security.
